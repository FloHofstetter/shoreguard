    auth ¶

    User-based authentication with service principals for Shoreguard.

    Two identity types: - Users: email + password → session cookie (Web UI) - Service Principals: API key → Bearer token (Terraform, CI/CD)

    Both carry a role: admin, operator, viewer (hierarchical).

    Three credential transports: 1. Authorization: Bearer <sp-key> header — API / Terraform / curl 2. sg_session cookie (HMAC-signed) — Web UI after login 3. ?token=<sp-key> query parameter — WebSocket connections
